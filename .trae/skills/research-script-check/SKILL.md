# 科研脚本检查

使用本技能为科研脚本添加标准化分步核对层，正确性以符合公式或预期效果为准，而不仅是无报错运行。

## 适用场景
- 用户要求逐步核对推导/公式
- 用户希望对长表格给出明确输出或抽检
- 用户希望形成可复用的科研脚本检查规范

## 标准流程
1. 复制原脚本为 *_check.* 版本。
2. 添加中文注释说明：
   - 每个代码块要实现的公式或效果
   - 如何通过输出进行核验。关键公式需严格检查对照参考材料中的公式全检（如生成IBP的种子）。table元素大于20时可抽检观察部分元素+可选脚本全检重要性质。
3. 每步后添加 Print 输出：
   - 给出紧凑的代表性表达式
   - 长输出使用 Take/UpTo 等抽取小片段
4. 能做公式核对时显式检查：
   - 输出 Simplify[计算结果 - 期望结果] 并期望为 0，其中期望结果为抄录自科研文献笔记材料或基于文献材料推理。
5. 若检查项目可通过脚本进行快速完全检查（不涉及解析式化简、积分等复杂计算），则进行完全检查并明确输出结论。
6. 部分检查项目通过可代入分子分母为1000以内整数的有理分数加速检验，避免代入过于简单的数导致偶然的相消。
7. 保持检查非破坏性：
   - 不改核心逻辑，只增补检查与输出
8. 如果检查中发现不符，需全部问题调试修正完成后，将修正内容回写到正式版代码文件。

## 输出清单
- 基础变换或指标平移：抽检 1 个
- 核心生成器/关系：各抽检 1 个
- 对称性或约束：做 Simplify[差值] = 0 的检查
- 过滤或筛选规则：给出应被置零的例子
- 范围或边界：输出推导后的范围
- 大型集合：显示前 3–5 项
- 关键派生集合：显示前 2–3 项
- 映射/目标列表：显示前 3–10 个编号

## 备注
- 计算足够快时（20s内）优先使用全检，其次确定性抽检（Take/UpTo），避免随机抽样。
- 输出保持简短且可读。
