# 代码注释规范（中文）

本技能用于约束所有 AI 编写或修改的代码必须提供清晰、简明的中文注释，便于后续维护与复用。

## 适用场景
- 用户要求新增或修改代码
- 用户希望提高可读性、可维护性
- 用户要求形成统一注释风格

## 注释要求
- **文件头部（必须）**：所有代码、脚本最开头必须详细注释本文件的用途、功能范围及核心逻辑。
- **模块实现思路（必须）**：每个模块（或主要代码块）需简要介绍实现思路，一般不超过三行。
- **模块级注释**：说明模块用途、依赖、输入输出、边界条件。
- **函数级注释**：说明参数、返回值、前置/后置条件、异常或特殊情况。
- **复杂逻辑**：给出思路与关键不变量；避免冗长重复。
- **安全**：不得记录或泄露密钥、账号、私有路径。
- **语言**：统一使用中文。

## 书写位置
- **文件首行**：放置详细的文件用途说明。
- **模块头部**：放置模块级注释及实现思路。
- **函数定义前**：放置函数级注释。
- **代码块上方**：需要时添加简短逻辑说明。

## 示例结构
1. **文件头**：详细描述文件作用。
2. **模块/代码块**：
   - 用途与依赖
   - **实现思路**（<3行）
3. **接口说明**：输入、输出、错误处理
3. 使用示例（可选，简短）

## 触发条件
- 当用户提出代码改动请求时必须遵循此规范
