# 科研文献调研与引用工作流 (Research Reference Workflow)

本技能定义了一套系统的寻找、筛选和管理参考文献的流程，特别适用于高能物理领域。

## 1. 准备阶段 (Preparation)

### 1.1 需求锚定
**第一步（关键）**：根据当前项目的论文草稿或研究计划，明确行文中需要引用的具体**细分领域**和**具体贡献点**。
*   *目的*：避免盲目收集，只针对实际写作需求寻找引文。

### 1.2 工作目录结构
在项目主目录下创建 `refbib` 文件夹，包含两个子目录：
*   `refbib/Reference_Sources` (文件A)：存放用于“参考其引文”的高价值综述或重要论文。
*   `refbib/Citation_Library` (文件B)：存放最终筛选出的目标引用文献（PDF及相关资料）。

## 2. 筛选“参考论文” (Selection of Reference Sources)

用于构建“文件A”的策略。通过交叉验证筛选出 5-10 篇高价值论文作为引文线索来源。

### 筛选标准
1.  **用户指定**：首先包含用户明确给出的核心论文。
2.  **细分方向关联**：针对特定细分方向（方向C），在 INSPIRE-HEP（优先）或 Google Scholar 中寻找。
3.  **时间与影响力**：
    *   **近期论文**（优先）：半年内发表，越新越好（确保覆盖最新进展）。
    *   **高引论文**：两年内发表且引用数 > 50 次，被引越多越好（确保权威性）。
    *   **知名作者**（辅助）：优先选择总引用数高、在该领域活跃的作者的论文。
4.  **相关度检查**：检阅论文内容，确保其确实涉及本项目所需的细分领域。若偏离则剔除并替换。
5.  **补充筛选**：若现有参考论文未覆盖所有细分领域，需针对性补充，并建立以细分领域命名的子文件夹。

### 标记管理
*   给每篇存入“文件A”的参考论文分配数字编号（如 [R1], [R2]...），便于后续追踪。

## 3. 目标引文分析与整理 (Citation Analysis)

在确定了“参考论文”后，在 `refbib` 下建立一个 TeX 文档（命名为 `citation_analysis.tex`），进行分析记录：

### 3.1 定向提取与排序
1.  **按需提取**：**不要**提取参考论文的所有引文。仅根据 **1.1 步** 确定的细分领域和贡献点，从参考论文中提取相关的文献。
2.  **分类整理**：按“细分领域”为章，“具体贡献”为节进行归类。
3.  **数据查询与排序**：
    *   查询每篇目标文献的被引次数（优先 INSPIRE-HEP，其次 Google Scholar，注明来源）。
    *   按被引次数**降序排列**（被引多的排在前）。
4.  **交叉引用统计**：记录该文献被哪几篇“参考论文”同时引用（使用编号 [R1], [R2] 标注）。被多篇引用的文献通常更重要。

### 3.2 内容总结与筛选
1.  **粗读总结**：对提取出的目标文献，粗读其摘要、简介和结论。
2.  **简明摘要（Note/Remark）**：**必须**极其简练，用**一句话**或**3-5个关键词**说明该文献的核心贡献或与本项目的关联（类似 BibTeX 中的 note 字段风格）。
    *   *Example*: "Introduces canonical differential equations."
    *   *Example*: "Method for IBP reduction using finite fields."
3.  **记录位置**：将总结记录在 `citation_analysis.tex` 中对应的条目下。
4.  **描述摘录**：简要记录各“参考论文”是如何描述该文献的，仅保留最关键的**半句话**，去除冗余上下文。
5.  **最终筛选**：综合评估后，标注“可忽略”的低价值或不相关论文。

## 4. 文献库构建 (Library Construction)

### 4.1 下载与存储
1.  **下载文献**：下载所有在 **3.2 步** 中未被标记为“可忽略”的目标文献（PDF）。
    *   **优先来源**：尝试下载已发表版本（Journal Published Version）。
    *   **Fallback**：若发表版无法下载（需付费/无权限），**必须**前往 INSPIRE-HEP 或 arXiv 下载预印本（Preprint）。确保获取到可阅读的全文 PDF。
2.  **存储位置**：存入 `refbib/Citation_Library` (文件B) 文件夹。
3.  **数量控制**：一般控制在 100 篇以内。如果超过此数量，需暂停并询问用户是否继续下载或进行二次筛选。

### 4.2 Bib 文件生成
基于 `citation_analysis.tex` 的整理结果，在“文件B”中构建最终的 `.bib` 文件。

1.  **获取 BibTeX**：
    *   **首选**：INSPIRE-HEP（搜索论文 -> Cite -> BibTeX）。
    *   **备选**：Google Scholar（点击引用 -> BibTeX）。
    *   **验证**：必须确保 Key 格式（`Author:Yearxxx`）和字段符合 `bib-management-standard` 技能规范。
2.  **文件组织**：
    *   按细分领域/具体贡献分块（使用 `%------Title------` 分隔）。
    *   在条目中添加注释，备注其具体贡献或重要性。
3.  **排除**：不录入被标注为“可忽略”的文献。
4.  **最终引用**：Bib 文件作为备选库，论文写作时根据行文逻辑按需引用（`\cite{Key}`）。
