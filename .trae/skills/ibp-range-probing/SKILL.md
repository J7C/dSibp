# IBP 指标范围试探

本技能用于系统化试探 IBP 指标范围，并记录每次约化的范围与截断积分信息，指导下一步扩展或收缩。

## 适用场景
- 用户要求“试探/扩大/收缩 IBP 指标范围”
- 用户要求“记录每次约化的指标范围与截断积分”
- 用户需要根据截断积分分布选择下一步扩展方向

## 存放目录
每次试探的约化结果results都复制回第一次约化结果复制回的目录下，文件夹名区分试探次数。
## 目标积分
- 项目所需的所有目标积分（主要是主积分及其动力学量偏导）以及 编号小于400的所有积分
- 或以用户要求为准
## 截断积分
检查 Kira 输出的`master`主积分编号（其中含因ibp关系不足而未约化的伪主积分），寻找其中的截断积分
**截断积分定义为 master中某一积分编号是上一行积分编号的3倍的积分** 其编号记为n。没有时记截断积分编号为0。
编号小于截断积分编号的积分大概率为真主积分。
## 未约化积分
编号小于截断积分编号的积分，称为未约化积分。

## 指标范围试探策略

### 1. 初始范围构建
1. **种子关系基准**：以生成 IBP 的种子关系式中所有指标 = 0 为基准。
2. **最小 IBP 范围**：找到 IBP 关系能成立的最小指标范围（如 `{-1,1}` 或 `{-2,2}` 等）。
3. **范围 A**：将上述最小范围的上下限都扩大 1，作为初步安全范围。
4. **微分方程范围 B**：
   - 若已选定主积分：对主积分求导数（在微分方程中出现），统计导数后出现的积分指标范围。
   - 若未选定主积分：以指标全 0 为基准求导数，记录 `a`、`b` 等指标的上下限均扩张1 作为范围 B。
5. **初始 IBP 范围**：取范围 A 与范围 B 的并集，作为第一次约化的指标范围。如果用户要求特定范围，则以用户要求为初始范围。

### 2. 逐步扩展与学习
1. **运行首次约化**：用初始范围生成 `ibp.kira` 与 `list`，运行 Kira 并记录约化数据。

3. **单步扩展**：
   - 每次只选择 **一类指标（如均命名为a_i则为一类）的一个边界（上限或下限）扩大 1**。   
   - 根据**指标增量选择策略**改变指标范围后重新生成 IBP 系统进行约化（运行Kira），并根据 3. 记录约化数据。
4. **指标范围扩张方向选择：上限或下限**：
   - 若目标积分中未约化积分的**某个指标正值较大**，则优先增加该类指标的**上限**。
   - 若目标积分中未约化积分的**某个指标负值较大**，则优先增加该类指标的**下限**。
   - **均以上两种情形出现时**，优先方向取 上限，此时开始记录扩张指标，此后所有上限扩张>=1后，优先方向改为 下限，直至所有下限扩张>=1后，优先方向改为 上限，以此类推。
5. **此后指标范围扩张选择策略**：
   - 若扩展后截断积分编号n**没有增加**，说明该边界扩展给出的或许不是最优先需要的关系式，称为扩张前的指标范围下的**无效扩张**（但可能和后续其他扩张组合在一起约化掉更多积分），应**保持本次扩张**，并**记录本次约化数据**，但在其他类指标向同方向扩张但没能增大截断积分编号前都不再扩张此类指标此方向限制(除非已遍历扩张此方向所有指标的)。同时，**再换一类指标或边界扩张**
   - 若截断积分编号增加，说明（在扩张前指标范围下）该扩展有效，**记录本次约化数据**。继续向该方向扩张该指标直至无效。
6. **迭代**：根据 4~5 不断扩张指标范围，直到满足以下所有条件：
   - 出现截断积分（编号是master中上一积分编号的3倍的积分）
   - 所有目标积分（项目所需，主要是主积分及其动力学量偏导）以及编号小于200的积分都能被约化（指标小于截断积分编号n）

### 3. 范围精简与验证
1. **完成目标后**：当目标积分全部可约化时，尝试**收缩**之前扩展但可能无效的边界。
2. **收缩测试**：将某个边界的上限/下限收回 1，重新约化。
3. **验证影响**：若收缩后目标积分仍可约化，且截断积分未增加，则说明该边界可永久收缩，继续记录此次约化的上下限与截断积分编号。
4. **记录最优范围**：经过扩展与收缩测试后，得到**最小必要指标范围**。

### 4. 记录与自动化
1. **日志文件**：每次约化均记录以下信息到约化生成系统所在目录下专门文件,勿覆盖此前实验记录（如 `index_range_log.txt`）：
  - 约化的指标范围
  - 本次约化扩张的所有指标及方向（上限/下限）
  - 已经进行了无效扩张的指标（直至同方向某类指标有效扩张后清空）
  - 截断积分编号n
  - 前 n+30 个master中的编号*
2. **决策依据**：日志文件用于指导后续的边界选择，避免重复无效尝试。
3. **使用自动化脚本选择扩展边界**：注意另开一个文件写此脚本，不要覆盖原来的；放于原约化脚本同一目录下。

### 5. 注意事项
- **数值约化**：除任选一个参量外，其他参量均任取较为一般的有理分数，从而加速约化；保留1个参量时为了避免触发kira bug。
- **FireFly 要求**：全数值代入时，**必须**开启 FireFly (`run_firefly: true`) 并添加辅助方程，详见 `kira-manager` 技能。
- **逐步扩大**：避免一次性扩大太多指标，以免系统过大导致 Kira 内存/时间爆炸。
- **与 DE 协作**：微分方程脚本（`de-manager`）应能读取同一日志，确保 DE 构建使用的范围与 IBP 范围一致。
